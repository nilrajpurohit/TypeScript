1. What TypeScript:
    TypeScript is a superset of JavaScript that adds features like static typing to improve code clarity, catch errors early, and enhance development. Here's a breakdown:
    - Static Typing: TypeScript allows you to define types for variables, function parameters, and return values, making it easier to catch bugs early. Example: let message: string = "Hello, world!";
    - Compile-to-JS: TypeScript needs to be compiled into JavaScript (via the TypeScript compiler tsc), allowing you to use TypeScript features while running standard JavaScript.
    - Type Inference: TypeScript automatically infers types when you don't explicitly define them, improving flexibility without sacrificing type safety. Example: let number = 42; // TypeScript infers 'number'
    - Better Tooling: TypeScript's type system enables better autocompletion, inline documentation, and code refactoring in IDEs and tools.
    - Advanced Features: TypeScript introduces features like interfaces, enums, generics, and decorators that aren't present in JavaScript.

2. Why TypeScript exists:
    The primary reason TypeScript exists is to catch errors at compile time rather than at runtime, which improves the development process by making code more predictable, easier to debug, and maintain. Let me break it down:
    - Catching Errors Early: TypeScript catches type-related errors at compile time, preventing runtime issues.
    - Better Developer Experience: Static typing and IDE support improve autocompletion and error detection.
    - Enhanced Readability & Maintainability: Explicit types make code clearer and easier to understand and maintain.
    - Improved Tooling: TypeScript offers better tooling for static analysis, refactoring, and code navigation.
    - Stronger Long-Term Code Health: The type system helps maintain and scale large codebases with fewer bugs.
    - JavaScript Superset: TypeScript extends JavaScript with types while remaining fully compatible with JavaScript.

3. How TypeScript Differs from JavaScript:
    TypeScript builds on JavaScript by adding features like static typing, interfaces, and access modifiers. Here's a breakdown:

    - Static Types: TypeScript introduces static typing, allowing you to define explicit types for variables and functions. JavaScript is dynamically typed, meaning types are determined at runtime.
    Example: let num: number = 42;  // TypeScript enforces 'number' type

    - Interfaces: TypeScript allows you to define interfaces to specify the shape of objects or functions, providing better structure and type safety. JavaScript doesn’t have a built-in concept of interfaces.
    Example:
    interface Person {
        name: string;
        age: number;
    }
    const person: Person = { name: "Alice", age: 30 };

    - Access Modifiers: TypeScript supports access modifiers (public, private, protected) to control the visibility and access to class members, enhancing encapsulation. JavaScript doesn’t have built-in support for access control.
    Example:
    class Employee {
        private salary: number;
        constructor(salary: number) {
            this.salary = salary;
        }
        public getSalary() {
            return this.salary;
        }
    }

4. TypeScript is Structural:
    In TypeScript, types are based on their structure (or shape), not their name. This means that two objects with the same shape are considered the same type, even if they have different names. Here's a breakdown:

    - Structural Typing: TypeScript uses structural typing, which means types are compared based on their properties and methods, not the names of the types themselves.
    Example:
    interface Person {
        name: string;
        age: number;
    }
    const person = { name: "Alice", age: 30 }; // This works as 'Person'
    const employee = { name: "Bob", age: 40, department: "HR" }; // This works as 'Person' too

    - Type Compatibility: If two types have the same shape, TypeScript considers them compatible, regardless of whether they are named the same.
    Example:
    interface Car {
        wheels: number;
    }
    const bike = { wheels: 2 }; // Works as 'Car' because the shape matches
    
    - Structural vs Nominal: Unlike languages with nominal typing (e.g., Java, C#), where types must have the same name to be compatible, TypeScript focuses on the structure of types, making it more flexible and allowing for better code reuse.
